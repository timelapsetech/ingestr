Sort and rename image sequences—the easy way

Ingestr is a focused macOS app for anyone who shoots hundreds or thousands of stills (time lapse, bursts, or card dumps) and needs them renamed and organized into clear, dated folders—without a spreadsheet or manual numbering. Built-in copy verification lets you optionally check that each file copied correctly, from a quick size check up to a full byte-for-byte pass.

Download for macOS Source on GitHub

Requires macOS 12 or later · Current release 1.3

Pick a source folder and an output folder, choose how files should be named, and start ingesting. The window stays responsive on large jobs, with a clear progress bar and per-step status while metadata is read and files are copied. Under Rename Options, set Copy verification to match how careful you want to be—default stays fast; stronger modes help ensure nothing was lost in transit.

What Ingestr does for you

The app is deliberately simple: it is built around sequence detection, consistent naming, and a predictable folder layout—so you spend less time housekeeping and more time editing.

Smart sequence detection

Groups images that belong together based on capture time, so multi-session card dumps can be split sensibly when you want them to be.

Auto rename from EXIF

Optional automatic names derived from the date in your files, with a consistent pattern for sequences and camera-original style suffixes.

Auto split sequences

When auto rename is on, detect large time gaps and start a new sequence—useful when one folder contains more than one shoot.

Extras folder

Very small groups of images are moved to an Extras area so your main sequences stay clean.

Extension filter

Limit processing to one type (for example JPG or RAW) and leave everything else untouched.

Add to existing

Append new shots to a sequence that is already on disk: finds the last number, matches padding, and continues without collisions.

Copy verification

Optional checks when files are copied to the destination: leave it off for the same speed as before, compare sizes for a light sanity check, or use full verification when you need confidence every byte matches.

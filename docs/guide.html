How to use Ingestr

Quick start

  1. Launch Ingestr on your Mac (macOS 12 or later).
  2. Source directory: Drag a folder onto the Source area, or click the zone to choose a folder. This is where your images (or nested folders of images) live.
  3. Output directory: Drag or choose where organized sequences should be written.
  4. Options: Set the file extension filter and rename options (explained below).
  5. Start Ingesting: The app shows progress and detail while it works. You can cancel an in-progress run.
  6. When finished: Use Open Folder in the completion alert to reveal the result in Finder.

Options explained

These match the labels in the app’s Rename Options section.

File extension
Enter an extension (for example jpg or raw) to process only files with that extension. Leave the field empty to include all files the app considers.
Auto Rename
When on, Ingestr uses EXIF date information from your images to build folder names and a consistent naming pattern. Typical folder name shape: YYYYMMDDXCO_ where the date comes from the image, X is a sequence index for that day, and CO marks camera originals. When off, you choose a Base Name yourself and files are numbered after that prefix.
Auto Split Sequences
Available when Auto Rename is on. Detects significant time gaps between shots and starts a new sequence instead of lumping everything into one run. Helpful when one folder contains images from more than one session.
Add to Existing
When enabled, Ingestr looks at files already in the destination that match the naming pattern, finds the highest existing number, and continues from the next value using the same zero-padding. Use this to add new images to an ingest you already ran without overwriting or duplicating numbers.
Copy verification
None (default) uses the same copy behavior as earlier releases—no extra checks. Full copies in a streaming pass while computing a SHA-256 digest, then reads the destination once to confirm the bytes match (strongest check; slower on very large batches). Size only compares file sizes after copy (minimal overhead; does not catch corruption when sizes still match).
Base Name
Shown when Auto Rename is off. This is the text prefix before the sequence number (for example MyShoot_).
Number Padding
How many digits to use for the numeric part (for example 4 digits → 0001, 0002, …). Keeps filenames sorting correctly in Finder and other tools.
Start Number
The first number to use for the sequence. Pair with Add to Existing when you need to align with work already on disk, or set manually when you want a specific starting index.

Output folder layout

With Auto Rename, Ingestr organizes output under a year folder, then per-sequence folders. Small or non-sequential groups may go to Extras. 

Output Directory/
└── YYYY/
    ├── YYYYMMDD1CO_/
    │   ├── YYYYMMDD1CO_0001.jpg
    │   ├── YYYYMMDD1CO_0002.jpg
    │   └── ...
    ├── YYYYMMDD2CO_/
    │   └── ...
    └── Extras/
        ├── YYYYMMDD-HHMMSS.jpg
        └── ...

Installation

Download Ingestr.zip (the signed app archive—not the “Source code” download on the release, which is only the repository). Unzip it, drag Ingestr.app into Applications, and launch. Grant folder access when macOS asks, so Ingestr can read your source and write the destination.
